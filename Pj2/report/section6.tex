\section{Simulation}

For the simulation part I used Icarus Verilog and GTKWave.  For each circuit I
made a small testbench and applied a few different inputs.  The output waveform
is saved in a VCD file, so it can be opened in GTKWave.

The tested parts are:
\begin{itemize}
  \item one bit full adder,
  \item four bit ripple carry adder,
  \item one bit 2 to 1 multiplexer,
  \item four bit 2 to 1 multiplexer,
  \item eight bit ripple carry adder,
  \item eight bit carry select adder.
\end{itemize}

\subsection{Delay Comparison}

In the ripple carry adder the carry must pass through each full adder.  Because
$t_{co}=8$ ns, the final carry of an 8 bit ripple adder is about
$8\times 8=64$ ns.  The last sum bit also needs the sum delay, so it is about
$7\times 8+10=66$ ns.

In the carry select adder, the lower four bits are calculated first.  At the same
time the upper four bits are calculated twice: one time with carry input 0 and
one time with carry input 1.  After the lower carry is ready, multiplexers choose
the correct upper result.  This delay is about $4\times 8+5=37$ ns.

So the carry select adder is faster, but it uses more hardware because it has two
adders for the upper four bits.
